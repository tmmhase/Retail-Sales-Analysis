# Retail-Sales-Analysis

## ongoing
## Project Overview

This project explores transactional retail sales data using PostgreSQL to uncover customer purchasing behavior, product category performance, and sales trends. The analysis covers the complete data workflow, including database design, data validation, exploratory analysis, and business-focused SQL queries.

The objective was to practice real-world SQL techniques commonly used by data analysts, such as filtering, aggregations, window functions, Common Table Expressions (CTEs), and customer segmentation analysis. By transforming raw sales records into actionable insights, the project demonstrates how SQL can be used to support data-driven decision making in a retail environment.

## Skills Demonstrated

- SQL Query Development
- PostgreSQL
- Data Cleaning and Validation
- Exploratory Data Analysis (EDA)
- Aggregate Functions
- Window Functions
- Common Table Expressions (CTEs)
- Business KPI Analysis

## Dataset

The dataset contains retail transaction records including:

- Transaction ID
- Sale Date and Time
- Customer ID
- Customer Demographics
- Product Category
- Quantity Sold
- Unit Price
- Cost of Goods Sold (COGS)
- Total Sale Amount

The data was imported into PostgreSQL and analyzed through a series of structured SQL queries designed to answer key business questions related to sales performance, customer behavior, and product demand.
